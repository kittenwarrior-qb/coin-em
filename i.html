01

Tổng quan dự án

Giải pháp dựng trên ba lớp: app lõi Odoo, module OCA mã nguồn mở, và phần phát triển tùy biến của CDS. Nguyên tắc xuyên suốt — code tùy biến chỉ nằm trong addons/custom/, không bao giờ sửa lõi Odoo hay module OCA.

App lõi Odoo

stock · sale · purchase · account · hr · maintenance — cài qua depends, không cần source riêng.

Module OCA (addons/oca · git submodule)

Mã nguồn mở, kéo về dạng submodule. KHÔNG sửa — chỉ depends và cấu hình.

Custom CDS (addons/custom · CODE Ở ĐÂY)

Umbrella satori_base + các module nghiệp vụ. Mọi tùy biến version-control nằm ở đây.

Cấu trúc thư mục

satori-erp/
├── addons/
│   ├── custom/          # Module CDS — CODE Ở ĐÂY
│   │   ├── satori_base/ # Umbrella: depends module custom + master data
│   │   └── ...          # Các module nghiệp vụ
│   └── oca/             # Git submodule OCA — KHÔNG SỬA
├── config/              # File cấu hình Odoo
├── docker/              # Dockerfile, nginx
├── docs/                # Tài liệu dự án (trang này)
├── tools/               # Script migrate, backup
├── tests/               # Integration test
├── .claude/             # Setup Claude Code AI pair-coding
├── .github/             # CI workflows, PR template, CODEOWNERS
├── docker-compose.dev.yml
├── docker-compose.prod.yml
└── Makefile

Ranh giới sửa code

Claude Code và mọi dev chỉ được sửa trong addons/custom/. Thư mục addons/oca/ là submodule — cập nhật qua make oca-update, không commit thay đổi tay vào đó.

Setup lần đầu — 5 bước

Clone repo + OCA submodules

Cờ --recursive kéo luôn submodule OCA. Quên thì chạy make oca-init sau.

git clone --recursive git@github.com:<org>/satori-erp.git
cd satori-erp

Tạo file .env từ template

Sửa POSTGRES_DEV_PASSWORD — chọn password bất kỳ cho local.

cp .env.example .env

Khởi động dev stack

Dùng image odoo:19 official, không cần build.

make dev-up

Tạo DB + cài module + master data

Cài umbrella satori_base, kéo theo mọi module custom, nạp master data (no demo). Cần state sạch sau này → make reset-db.

make init-db
make dev-restart

Mở trình duyệt

Vào http://localhost:8069 → trang login.

Lưu ý môi trường

Database Manager bị tắt (list_db=False, dbfilter=satori_dev) — vào thẳng login, không chọn DB.

Cần tiếng Việt locale/font, HĐĐT, báo cáo TT99… thì make dev-build (xem ghi chú trong docker-compose.dev.yml).

05

Tạo một module — end-to-end

Tách 2 chuyện hay bị gộp: dùng module có sẵn ≠ code module custom. Cài module core/OCA chỉ cần khai depends; chỉ khi muốn tùy chỉnh mới dựng module riêng trong addons/custom/.

Dùng module có sẵn

Core Odoo: thêm tên kỹ thuật vào depends, chạy make update. Source có sẵn trong Odoo.

OCA: đã ở addons/oca/ qua submodule — chỉ thêm vào depends. Nhớ đúng nhánh 19.0 và test trước (OCA port theo từng version).

Code module custom

Khi cần model/field/logic mới hoặc sửa hành vi module có sẵn — dựng module riêng, prefix satori_*.

Mọi tùy chỉnh phải là code — không sửa lõi Odoo hay file OCA (upgrade là mất, team không merge được).

Mỗi tùy chỉnh rơi vào đúng một ô

Phân biệt cốt lõi: tạo data (dòng dữ liệu mới) khác hẳn inherit (mở rộng cấu trúc có sẵn).

Trình tự dựng — 7 bước

Khung manifest

Tạo __manifest__.py (khai depends) + __init__.py. Module rỗng nhưng cài được.

Định nghĩa model Python

Model mới hoặc _inherit thêm field. Field là cột dữ liệu — phải tồn tại ở model trước khi hiện ra view.

Khai quyền CSV · đừng quên

Mọi model mới bắt buộc có dòng trong ir.model.access.csv. Quên là lỗi, hoặc tệ hơn — model lộ cho mọi user.

Hiển thị lên giao diện XML

Field chỉ hiện khi nhét vào view: ir.ui.view + inherit_id + <xpath>, hoặc view mới cho model mới.

Master data đi kèm XML

Dùng <record> trong data/. Luôn ref bằng External ID, không hardcode số.

Liệt kê file đúng thứ tự manifest

Trong data: security → views → data. Sai thứ tự → lỗi External ID not found.

Load · verify · đồng bộ deploy

Đổi model/data/security → make update M=satori_returnable. Chỉ sửa view → --dev=xml tự reload. Verify bằng External ID, rồi commit; đồng nghiệp git pull → make reset-db → make init-db.

Hai mảnh dễ sai nhất

Thêm field "Mã kho nội bộ" là hai tầng inherit: Python tạo field ở model, XML hiện field ở view. Thiếu tầng Python → lỗi Field does not exist.

# models/stock_warehouse.py — tạo field (tầng model)
from odoo import models, fields

class StockWarehouse(models.Model):
    _inherit = 'stock.warehouse'
    x_satori_code = fields.Char("Mã kho nội bộ")
<!-- views/stock_warehouse_views.xml — hiện field (tầng view) -->
<record id="view_wh_form_satori" model="ir.ui.view">
  <field name="model">stock.warehouse</field>
  <field name="inherit_id" ref="stock.view_warehouse"/>
  <field name="arch" type="xml">
    <xpath expr="//field[@name='code']" position="after">
      <field name="x_satori_code"/>
    </xpath>
  </field>
</record>
# security/ir.model.access.csv — bắt buộc với model mới
id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink
access_returnable_user,returnable.user,model_satori_returnable,base.group_user,1,1,1,0

Code là nguồn chân lý, UI chỉ là nháp. Mọi thứ quan trọng phải tái tạo được từ git clone + make init-db.
Thứ chỉ sống trong DB của một người là nợ kỹ thuật.
